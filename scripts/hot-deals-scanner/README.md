# Hot Deals Scanner v2 — детектор лотов ниже рынка (перекуп)

Бот сканирует Avito (Москва+МО) по 5 семействам Mac, находит объявления **ниже
живого рынка** с **чистым состоянием** и шлёт их в Telegram.

Активный файл: **`scanner_v2.py`** (`scanner.py` — старая v1, не используется).

## Как он принимает решение

1. **Живой рынок, а не снимок базы.** На каждое семейство грузится несколько
   страниц выдачи (`SCAN_PAGES_PER_FAMILY`, по умолчанию 3). Из них строится
   распределение цен сопоставимых лотов (`семейство+чип+экран+RAM+SSD`) — то, что
   выставлено **прямо сейчас**. Лот сравнивается с **медианой** и **P20** этой
   живой выборки, а не с замороженным `min_price` из `avito-prices.json`.
   → больше нет «бот зовёт низом рынка лот, ниже которого висит десяток дешевле».

2. **Гейт состояния.** Для лота-кандидата открывается объявление и **полное
   описание** анализируется в `common/condition.py`: дефекты (вздутие, залитие,
   замена матрицы/платы, вскрытие, iCloud/MDM, запчасти…), здоровье АКБ (%),
   циклы. Вердикт `reject` → лот **не отправляется вообще**; `suspect` (косметика,
   АКБ 80–84%, повышенные циклы) → топится в скоринге.
   → больше нет «все присланные аппараты с проблемами».

3. **Скоринг перекупа** (`score_deal`): якорь — выкупная цена и запас маржи к
   живой медиане; чистое состояние поднимает, подозрительно дешёвое без
   подтверждения чистоты (гарантия/чек/комплект) — топится (антифрод −50) и
   никогда не уходит в realtime-алерт.

Порог realtime-алерта — `MIN_NOTIFY_SCORE` (75). Лоты 40–74 копятся в дайджест
(`scanner_v2.py --digest`, крон ~20:00 МСК).

## Тюнинг (env, у всех есть дефолты)

| Переменная | Дефолт | Смысл |
|---|---|---|
| `SCAN_PAGES_PER_FAMILY` | 3 | страниц выдачи на семейство (больше = надёжнее медиана, дольше скан) |
| `MIN_COMPS` | 6 | минимум живых сопоставимых для уверенного алерта |
| `MIN_MARGIN` | 0.10 | насколько ниже медианы, чтобы считать сделкой (запас под перепродажу) |
| `SCAM_FLOOR` | 0.55 | ниже этой доли медианы без чистоты → «подозрительно дёшево» |
| `BUYOUT_FACTOR` | 0.80 | выкуп-цель = доля от живой медианы (если нет курируемой) |
| `BATTERY_HARD` / `BATTERY_SOFT` | 80 / 85 | пороги % здоровья АКБ (reject / suspect) |
| `CYCLES_HARD` / `CYCLES_SOFT` | 1000 / 500 | пороги циклов (reject / suspect) |
| `MIN_NOTIFY_SCORE` / `DIGEST_MIN_SCORE` | 75 / 40 | пороги realtime / дайджеста |

Если присылает мало — снизь `MIN_MARGIN` (напр. 0.08) или `MIN_NOTIFY_SCORE`.
Если проскакивает шум — подними `MIN_MARGIN`/`MIN_COMPS` или `SCAM_FLOOR`.

## Запуск и деплой

```bash
# Тесты логики (без сети):
python3 scripts/hot-deals-scanner/test_logic.py

# Один проход:
python3 scripts/hot-deals-scanner/scanner_v2.py

# Дайджест (крон ~20:00 МСК):
python3 scripts/hot-deals-scanner/scanner_v2.py --digest
```

Прод крутится на VPS (Beget). Чтобы обновления вступили в силу — на VPS:
`git pull` и перезапустить процесс/таймер сканера. Новых зависимостей нет
(`condition.py`/`market.py` — только стандартная библиотека).
