# TG-leads — монитор Telegram-чатов (продавцы Mac → лиды на торг)

Read-only мониторинг целевых чатов: ловит посты «продаю MacBook/iMac…», фильтрует
мусор/дефекты/Intel/год, и кладёт лид в очередь бота-охотника
(`negotiation-queue.json`) — ты видишь его с кнопкой **«▶️ Веду торг»** и ссылкой
на сообщение продавца. Авто-рассылок нет (низкий риск бана) — пишешь продавцу сам.

## Файлы
- `target-chats.txt` — список чатов под мониторинг (правится вручную).
- `monitor.py` — сам монитор (ядро тестируется офлайн: `test_monitor.py`).
- `login.py` — разовая авторизация (создаёт сессию).

## Настройка (один раз)
1. **API-ключи:** зайди на https://my.telegram.org → *API development tools* →
   создай приложение → получишь `api_id` и `api_hash`.
2. **Зависимость:** `/opt/bestmac/.venv/bin/pip install telethon`
3. **Авторизация (интерактивно, нужен код из Telegram):**
   ```bash
   cd /opt/bestmac
   TG_API_ID=12345 TG_API_HASH=abc... .venv/bin/python scripts/tg-leads/login.py
   ```
   Введёшь телефон и код → появится `bestmac.session`.
4. **Ключи в .env** (чтобы сервис их видел):
   ```
   TG_API_ID=12345
   TG_API_HASH=abc...
   ```
5. **Запуск сервиса** (даю команду установки, когда сессия будет создана):
   `bestmac-tgmonitor.service` → `/opt/bestmac/.venv/bin/python scripts/tg-leads/monitor.py`

## Проверка ядра без сети
```bash
python3 scripts/tg-leads/test_monitor.py
```

## Важно
- Используй **отдельный** Telegram-аккаунт (не основной) — на случай ограничений.
- Только чтение публичных чатов. Массовые авто-DM не делаем — это триггер бана.
- Лиды-перекупщики в TG пока не отсеиваются по отзывам (в чате их нет); фильтр
  по отзывам работает на Avito-сканере.
